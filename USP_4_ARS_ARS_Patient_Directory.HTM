2.1 Application Overview
 ...
CMVK-0114 -163
The patient directory is the data management tool of the EBW systems, and all other systems using the EBW as a platform.
Patient Directory displays the Patient data that exists in the system, and also allows the user to work with similar data on other systems in the hospital network.
Patient data can be studies and images from multiple modalities (CT, MR, PET etc.)
Patient Directory allows easy and fast selection of the data and serves as a GUI platform for data operations such as load, copy, delete and modify.
All medical systems have some kind of user interface to allow such operations and the ease of use of the system is greatly influenced by the behavior of patient directory
 ...
CMVK-0114 -164
2.2 Clinical Background and Purpose
 ...
CMVK-0114 -165
The patient directory is the patient browser of the system and it is crucial that it will provide the best possible worklfow in order to load data into the various applications.
In order to achieve that, patient directory must provide information regarding the patients & DICOM data, as well as intuitive browsing and selection capabilities.
Patient directory provides the user interface for:
browsing between DICOM nodes in order to locate the required patient
database management such as copy & delete
user interface for loading data into the applications
Basic review capabilities of DICOM (preview & quick review) & non-DICOM (file viewer) data
interface for Recording CDs / DVDs
Patient Directory also serves as a 'home page' for the EBW and from it activation of Queue Manager, System Preferences and Logout from the system is being made.
